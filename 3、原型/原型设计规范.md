## 二、设计原则

### 2.1 云阿米巴核心原则

| 原则 | 说明 | 示例 |
|------|------|------|
| **流量优先** | 所有设计必须考虑"流量池"入口 | 首页突出流量数据 |
| **利益显性化** | 清晰展示"收益"、"分润"、"提现" | 我的页面突出收益 |
| **极简主义** | 去除冗余，专注变现与增长 | 减少无关功能入口 |

### 2.2 用户体验原则

| 原则 | 说明 |
|------|------|
| **一致性** | 相同功能保持相同的交互方式 |
| **反馈性** | 操作后必须有明确的反馈 |
| **可控性** | 用户可以撤销或取消操作 |
| **简洁性** | 界面简洁，减少认知负担 |
| **容错性** | 防止用户犯错，提供错误恢复 |

## 五、交互规范

### 5.1 加载状态

#### 骨架屏 (必须)

```
┌─────────────────────────────────────────────────────────────────┐
│  ████████████                                                   │
│  ████████████████████████████████████                          │
│  ████████████████████████████████████████████████████          │
├─────────────────────────────────────────────────────────────────┤
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ ████████  │  ████████████████████████████████████████   │   │
│  │           │  ████████████████████████████████████       │   │
│  │           │  ████████████████████████                   │   │
│  └─────────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────┘
```

**规则：**
- ✅ 必须显示骨架屏，严禁白屏
- ✅ 骨架屏形状与实际内容一致
- ✅ 骨架屏使用动画效果（闪烁/渐变）

#### 加载指示器

| 场景 | 组件 | 说明 |
|------|------|------|
| 页面加载 | 骨架屏 | 首次加载 |
| 列表加载 | 下拉刷新 + 上拉加载 | 列表数据 |
| 按钮操作 | Loading 按钮 | 提交操作 |
| 局部刷新 | Spin | 局部数据刷新 |

### 5.2 转场动画

| 场景 | 动画 | 时长 |
|------|------|------|
| 页面进入 | 从右滑入 | 300ms |
| 页面退出 | 向右滑出 | 300ms |
| 弹窗打开 | 从下滑入 + 淡入 | 250ms |
| 弹窗关闭 | 向下滑出 + 淡出 | 200ms |
| Tab 切换 | 淡入淡出 | 200ms |

### 5.3 反馈机制

| 场景 | 反馈方式 | 说明 |
|------|----------|------|
| 操作成功 | Toast 成功 | 绿色图标 + 文字 |
| 操作失败 | Toast 失败 | 红色图标 + 文字 |
| 加载中 | Toast 加载 | 加载图标 + 文字 |
| 确认操作 | Modal 确认 | 标题 + 内容 + 按钮 |
| 表单错误 | 输入框标红 + 错误提示 | 实时验证 |

### 5.4 手势操作

| 手势 | 操作 | 说明 |
|------|------|------|
| 下拉 | 刷新列表 | 列表页面 |
| 上拉 | 加载更多 | 列表页面 |
| 左滑 | 显示操作按钮 | 列表项 |
| 长按 | 显示更多选项 | 图片、内容 |
